# Plan Directory Guide

## Role
- Detailed build plan, architecture decisions, data plan, testing plan, prompt/eval plan, and delivery phases.

## How To Use
- Treat these files as design intent and review context.
- Check them when making changes that could violate earlier architecture decisions.

## Editing
- Keep these docs consistent with the actual codebase.
- If implementation intentionally changes a major architectural assumption, update the plan rather than leaving stale guidance behind.
