# Prompts Directory Guide

## Role
- Product-planning prompt inputs and workflow notes that informed the current implementation.

## Usage
- Read these when refining product behavior, evaluation criteria, or implementation scope.
- Do not assume files here are runtime prompts unless the code explicitly loads them.

## Editing
- Keep prompt intent clear and specific.
- If product behavior diverges from these docs, update the docs or the implementation intentionally rather than letting them drift silently.
